ENVIROMENTS

Se crea un ambiente por cada proyecto. Cada ambiente tiene multiples variables. Estas variables sirven para que cuando probemos endpoints, podamos cambiar de servidor (pasar de local, develop, etc).

Crear un nuevo enviroment: En el apartado izq click en enviroments -> agregar un nuevo enviroments -> 	asignarle el nombre del proyecto.
	En la tabla: 
		variable: nombre del servidor (local, develop, etc)
		initial value: la url correspondiente.

Para cambiar de enviroment: En la parte sup der de postman -> Seleccionar determinado enviroment.

Para usar una variable de un enviroment en los endpoint se coloca:
	{{nombreVariable}}


AUTORIZACION

Se usa cuando vamos a ingresar un token que solicita un servicio externo o nuestra propia seguridad
que implementamos por ejemplo con spring security + jwt.

*Estando en una peticion* -> click en el apartado 'authorization' -> elegir el tipo de autorización
	en la opcion 'Auth type' (lo común será seleccionar: 'Bearer Token') después se ingresa el tokan.


WORKSPACES

Es útil para tener aislado todas las collections que tienen que ver con el mismo proyecto pe una collection
para la api externa y otra collection para la api que uno esta desarrollando.

Crear workspace: parte sup der de postman -> workspaces -> créate workspaces -> blank workspaces.
