

crear Run Configuration's fijos ------------------------------

El botón verde de “Run” solo se activa si intelliJ detecta una clase ejecutable en el archivo actual.
Por eso solamnete cuando estamos en la clase principal del proyecto vemos el botón de Run, porque ahí 
IntelliJ detecta el main.

Para poder correr el proyecto sin importar el archivo en el que nos encontremos: Arriba (donde normalmente 
	aparece el botón verde) -> clic en el menú desplegable -> seleccionar "Edit Configurations" ->
	presionar + (Add New Configuration) -> Selecciona application -> Name: nombre del proyecto;
	Main class: seleccionar la clase principal -> Dale Apply y luego OK.

Se pueden crear configuraciones adicionales para cosas como:
Run con perfiles específicos (--spring.profiles.active=dev).
Run pruebas unitarias.



problemas ----------------------------------

Se tuvo un problema al momento de hacer un despligue automatico, al principio se tenia un application.yaml
pero al cambiarlo por su equivalente en application.properties, resulta que su codificación cambia de 
utf-8 a iso-8520-1 y al hacer el despliegue este solo acepta archivos con la codificación utf-8.
Solución: menú file -> settings -> apartado izq editor -> file encodings -> un poco hasta abajo en donde
dice: default encoding for properties files -> seleccionar utf-8 -> apply -> ok.
