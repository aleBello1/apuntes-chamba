Descargar o cambiar el  jdk: En la parte sup der -> click en el engrane -> Project structure
	-> Project settings -> Project -> SDK -> download jdk.

Buscar todos los lugares donde se manda a llamar un método: ir al lugar donde se define el método -> 
	click der sobre el nombre -> find usages.
	- Dar un click para ver una previsualización.
	- Dos clcks para que nos lleve a esa clase.

Para generar un jar utilizando la interfaz grafica de intelliJ: en la parte der dar click en el button the 	Maven y dar doble click sobre todas las siguientes opciones:
	- clean -> Elimina el directorio target/.
		Esto implica que se eliminen: Los archivos compilados (.class), Los JAR/WAR generados y 
		y archivos temporales de la compilación.
		Asegura que la próxima compilación empiece desde cero, evitando que queden restos de 
		compilaciones anteriores que puedan causar errores o comportamientos raros.
	- compile -> Compile el código fuente del proyecto, pero sin generar paquetes (.jar, .war, etc.) 
		ni ejecutar tests.
	- install -> Compilar el código, Ejecutar las pruebas unitarias y empaquetar el resultado 
		en un .jar o .war.


Atajos de teclado (shortcuts) ---------------------------------------

Aquí viene un pdf donde vienen todos los shortcuts de intelliJ:
	https://rubentejera.com/atajos-de-teclado-shortcuts-para-intellij-idea/

Agregar o modificar un shortcut: presionar ctrl + alt + S -> apartado keymaps -> escribir en el buscador el 
	nombre del shortcut a editar -> click der sobre el shortcut -> add keyboard shortcut.
	Esto se uso porque se tenia una lap donde para comentar multiples linea de código se tenia que usar
	el carácter '/' pero del teclado numerico, asi que se modifico. *****

Busca una palabra en todo el proyecto: control + shift + f.	

Duplicar la linea actual: ctrl + d. duplicate current line.

Convertir en mayuscular o minúsculas toda la palabra seleccionada:  ctrl + shift + u

Crea un multicursor en todas las coincidencias de la palabra seleccionada: ctrl + shift + alt + J

seleccionar una ocurrencia una por una de la palabra seleccionada: alt + J. Selection for next ocurrence.

problemas -------------------------------------------------

En ocasiones no esta habilitado el button de correr proyecto: en la parte sup der darle click en el button
	de notificaciones.
	Si no queda con eso, preguntar cual jdk debe estar instalado en el SO y no en el intelliJ

Solución al problema 'de la nada empezo a fallar mi proyecto de kotlin + spring boot, los tipos de datos y 
clases me marcaron que no están importados pero si lo estan': esto sucede cuando la indexación o el build de Gradle/Maven se rompe, y marca mal los imports aunque estos esten correctos. En intelliJ irse a menú File -> Invalidate Caches / Restar -> seleccionar Invalidate and Restart.



terminales -----------------------------------------------

Sacar y Elegir una terminal: En la parte inf. izq se muestra el logo de la terminal -> al darle click 
	se muestra una ventana que contiene una terminal por defecto.
	Para cambiarla se da click en la flecha hacia abajo dentro de esa ventana y se elige la git bash.

Los comandos que se ejecuten en esta terminal son equivalentes a los que se podría hacer con la interfaz 
grafica de intelliJ. *******
