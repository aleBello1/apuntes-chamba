////////////////////////////////////GLOSARIO/////////////////////////////////////////

Aplicación: conjunto de objetos, campos y otras funciones para cubrir un proceso empresarial.

organización: entornos para crear u ejecutar aplicaciones de Salesforce. Pe el trailehead
	Playground es un tipo de organización.

Trailhead Playground: Entorno seguro donde puede practicar las habilidades que está adquiriendo antes de aplicarlas en su trabajo real.

Objetos: son tablas en la base de datos de Salesforce que almacenan un tipo de información en particular.
	Tipos de objetos:
	- Objetos estándar: son objetos que están incluidos en Salesforce. 
	- Objetos personalizados son objetos que se crean para almacenar información 
		 específica de su empresa o sector.

Tipos de campo:
- Sistema: campos de solo lectura que ofrecen información sobre el sistema como, por ejemplo, 
	la fecha de creación del registro o la fecha de la última modificación.
- Nombre: Todos los registros necesitan nombres para poder distinguirlos.
- personalizado. Cuando se crea un campo personalizado, al nombre del campo se le agrega el 
	postfijo para identificarlo: __c.

Tipos de datos:
- Casilla: los campos de casilla son perfectos para los campos que requieren un simple 
	"sí" o "no".
- Date y DateTime.
- Formula: contiene un valor que se calcula automáticamente en función de una fórmula que 
	usted crea.

Tipos de relaciones:
- relaciones de búsqueda. Es como una relación de uno a muchos pero en si se usa cuando desde
	un objeto queremos buscar los registros que tiene asociados.
- relación principal-detalle. Es similar a una relación uno a uno, pero se hace mucho incapia 
	en que el objeto detalle depende totalmente del objeto principal.

ficha personalizada: Permite que se pueda agregar un objeto personalizado a una aplicación.

BOTONES  Y VINCULOS PERSONALIZADOS

botones personalizados: permiten conectar usuarios con aplicaciones externas.
	Tipos de botones:
	- Botón de lista: aparece en una lista relacionada en una página de registro de objeto.
	- Botón de página de detalles: aparece en el menú de acciones del panel de aspectos 
		destacados de una página de registro.

Crear un button personalizado: En la parte superior der click en el logo de engrane -> apartado 
	object manager -> click en determinado objeto -> en apartado izq que diga buttons,
	links and actions -> new button or link -> asignar un nombre 
	- button de lista.
	- button de pagina link (para poner enlaces externos)
	- button de detalle. Se usa cuando se usaran los datos del registro en un enlace, sea 
		externo o no.
	-> ingresar el link -> save y ok.
	El button no aparecerá hasta que se agregue.

Agregar el button: En la parte superior der click en el logo de engrane -> apartado 
	object manager -> click en determinado objeto -> en apartado izq que diga page layouts
	-> seleccionar determinado layout 
	- Agregar un button de lista: Presionar fields -> Desplazarse entre los campos y 
		seleccionar en determinado el símbolo de engrane -> ampliar el apartado de 
		buttons -> pasar el tipo de button llamado Audit guidelines -> ok -> sabe.
	- Agregar el button con enlace externo: Presionar custom links -> arrastrar determinado
		button al apartado de custom links.
	-  Agregar un button de detalle: Presionar buttons -> arrastrar determinado button al 
		apartado de custom buttons.

Vinculos personalizado: Son enlaces con una dirección URL externa.
	A los vínculos personalizados se les puede pasar datos en los parámetros de un enlace.

Comprobar la creación de button personalizado: En app lancher buscar determinada app -> click
	en determinado apartado (determinado objeto) -> Entrar a cualquier registro.
	- button de lista: Presionar en apartado related: Desplazarse hasta determinado campo
		-> ahí aparecerá con el nombre que le dimos.
	- button con enlace externo: Presionar en apartado details -> bajar y en apartados de 
		custom links aparecerá.
	- button de detalle: En la parte sup derecha de la pagina -> click en la flecha hacia 
		abajo -> nombre del button personalizado. 


ACCIONES RAPIDAS

Las acciones permiten a sus usuarios realizar tareas rápidamente, como crear registros, registrar llamadas, enviar correos electrónicos y muchas otras cosas. 

Tipos:
- Acciones específicas de objetos: permiten a los usuarios crear o actualizar registros 
	rápidamente, registrar llamadas, enviar correos electrónicos y mucho más en el contexto
	de un objeto concreto.
	Tipos:
	- créate
	- update
	- log a call: permiten a los usuarios introducir notas acerca de llamadas, reuniones u
		otras interacciones relacionadas con un registro específico.
	- Acciones que invocan componentes Lightning, flujos, páginas de Visualforce o 
		aplicaciones de lienzo.
	- send email.
- acciones globales: Las acciones globales están en un formato especial propio, conocido como 
	el formato global del publicador. No está asociado a ningún objeto y rellena el menú de 
	acciones globales en Lightning Experience. Si un formato de página de objeto no está 
	personalizado con acciones, las acciones de estas páginas de registro de objeto se 
	heredan desde el formato de publicador global.

Crear una acción específica de objeto: En la parte superior der click en el logo de engrane
	-> apartado object manager -> click en determinado objeto -> apartado izq y click donde
	diga Buttons, Links, and Actions -> new action -> seleccionar tipo de accion.
	- créate a record.
	-> seleccionar el objeto que ejecuta la accion -> asignarle nombre -> sabe.
	Editor de formato de acción: personalización de los campos que los usuarios deben 
		rellenar para completar la acción.
		Cada campo de este formato de acción tiene un asterisco de color rojo, lo que 
		indica que es un campo obligatorio. Los campos obligatorios se agregan a un 
		formato de acción de forma predeterminada cuando crea la acción. Si retira un 			campo obligatorio de un formato de acción, los usuarios no pueden completar la 
		acción correctamente.
	La accion aun no estará disponible para los usuarios.

Hacer que la accion este disponible para los usuarios: En la parte superior der click en el logo
	de engrane -> apartado object manager -> click en determinado objeto -> apartado izq y 
	click donde diga Page Layouts -> seleccionar determinado layout -> en este punto 
	estaremos en formato de pagina.
	Formato de pagina:
	- quick actions: controla qué acciones aparecen en páginas de registros en la interfaz 
		de usuario de Salesforce Classic.
	- actions: controla qué acciones aparecen en páginas de registros en Lightning 
		Experience y en la aplicación Salesforce.

FORMATO COMPACTO

Forma de definir qué campos de un registro se muestran de manera resumida en ciertas partes de la interfaz.
Sirven para que el usuario vea la información más importante de un registro sin tener que abrirlo completo.
Cuando crea un objeto personalizado, se asigna automáticamente a un formato compacto predeterminado del sistema, que solo tiene un campo: el nombre del objeto.

Crear un formato compacto para un objeto: En la parte superior der click en el logo de engrane
	-> apartado object manager -> click en determinado objeto -> en los apartados izq 
	click en compact layouts -> new -> asginar nombre al formato compacto y agregar los 
	campos de interés y sabe.

Establecer determinado formato compacto como el formato Compacto principal: click en el 
	button compact layout assignment -> edit assignment -> seleccionar determinado formato
	compacto -> sabe.

CAMPOS RESUMEN

Mientras los campos de fórmula calculan valores con campos de un único registro, los campos de
resumen calculan valores a partir de una serie de registros relacionados. Estos campos se usan
en una relación uno a muchos (principal - detalle).
Tipos de resumen:
- count
- sum
- min
- max

Crear campo resumen: En la parte sup derecha buscar el logo de settings -> opción setup -> 
	apartado object manager -> click en determinado objeto -> clic en Fields & Relationships
	-> New -> Roll-Up Summary -> next -> asignar nombre al campo -> seleccionar el objeto 
	que representa a muchos (en la relación uno a muchos) -> elegir el tipo de resumen y el 
	campo sobre el que recaerá la accion -> puro next y save.


REGLAS DE VALIDACION

verifican que los datos introducidos por usuarios en registros cumplen 
los estándares que especifica antes de poder guardarlos. Una regla de validación puede contener 
una fórmula o expresión que evalúa los datos en uno o más campos y ofrece un valor true o false.
Las reglas de validación también pueden incluir mensajes de error para mostrar a los usuarios cuando introducen valores no válidos basándose en criterios específicos.
Puede crear reglas de validación para objetos, campos, miembros de campaña o eventos clave de casos.

Crear regla de validación para un campo especifico: En la parte sup derecha buscar el logo de 
	settings -> opción setup -> apardato object manager -> click en determinado objeto
	-> apartado izq que diga validation rules -> new  -> asginar nombre a la regla, la 
	formula que si se rompe se lanza el mensaje, el mensaje de error -> save.

APEX

lenguaje de programación orientado a los objetos y con parámetros de asignación rigurosa que permite a los desarrolladores de Salesforce agregar lógica empresarial a eventos del sistema, como clics de botón y actualizaciones, a los registros relacionados.

Crear un nuevo proyecto de Salesforce y conectarlo a su organización de Trailhead Playground:
	*En vsc* ctrol + shift + p -> buscar y seleccionar SFDX: Create Project -> standard
	-> ingresar el nombre del proyecto -> ingresar la carpeta donde se estará el proeycto..

Para desarrollar con ápex, de acuerdo a la doc oficial, se recomienda tener vsc y la extensión
Salesforce Extension Pack (expanded). *

sf org login web -d -a nombreOrganizacion -> Para autenticarte en una organización de 
	Salesforce desde el navegador y configurarla con roles especiales dentro del CLI.

sf project deploy start -> Sirve para desplegar (deploy) el metadata de tu proyecto local 
	hacia una organización de Salesforce usando Salesforce CLI.
	Toma el contenido del proyecto Salesforce (normalmente la carpeta force-app/) y lo 
	envía a una org destino (scratch org, sandbox o producción).

.forceignore -> Para decirle a Salesforce CLI qué archivos o carpetas NO debe incluir cuando 
	haces operaciones como deploy, retrieve o convert de metadata.

sf apex generate class --name nombreClase --output-dir force-app/main/default/clases -> Para 
	crear una clase de ápex.
	Esto crear dos archivos: uno con extensión .cls y otro con extencion .cls-meta.xml
	en la ruta force-app/main/default.

Crear un script anonimo: es código de Apex que no se almacena en los metadatos, pero puede 	compilarse y ejecutarse.
	Crear un archivo con extensión .apex en la carpeta: scripts/ápex
