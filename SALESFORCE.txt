////////////////////////////////////GLOSARIO/////////////////////////////////////////

Aplicación: conjunto de objetos, campos y otras funciones para cubrir un proceso empresarial.

organización: entornos para crear u ejecutar aplicaciones de Salesforce. Pe el trailehead
	Playground es un tipo de organización.

Trailhead Playground: Entorno seguro donde puede practicar las habilidades que está adquiriendo antes de aplicarlas en su trabajo real.

Objetos: son tablas en la base de datos de Salesforce que almacenan un tipo de información en particular.
	Tipos de objetos:
	- Objetos estándar: son objetos que están incluidos en Salesforce. 
	- Objetos personalizados son objetos que se crean para almacenar información 
		 específica de su empresa o sector.

Tipos de campo:
- Sistema: campos de solo lectura que ofrecen información sobre el sistema como, por ejemplo, 
	la fecha de creación del registro o la fecha de la última modificación.
- Nombre: Todos los registros necesitan nombres para poder distinguirlos.
- personalizado. Cuando se crea un campo personalizado, al nombre del campo se le agrega el 
	postfijo para identificarlo: __c.

Tipos de datos:
- Casilla: los campos de casilla son perfectos para los campos que requieren un simple 
	"sí" o "no".
- Date y DateTime.
- Formula: contiene un valor que se calcula automáticamente en función de una fórmula que 
	usted crea.

Tipos de relaciones:
- relaciones de búsqueda. Es como una relación de uno a muchos pero en si se usa cuando desde
	un objeto queremos buscar los registros que tiene asociados.
- relación principal-detalle. Es similar a una relación uno a uno, pero se hace mucho incapia 
	en que el objeto detalle depende totalmente del objeto principal.

ficha personalizada: Permite que se pueda agregar un objeto personalizado a una aplicación.

Crear nuevo Trailhead Playground: Haga clic en su fotografía de perfil en la esquina 
	superior derecha de Trailhead -> clic en Hands-On Orgs (Organizaciones prácticas) ->
	clic en Create Playground (Crear Playground). El Playground puede tardar unos minutos
	en crearse.

Abrir determinado Trailhead Playground: Haga clic en su fotografía de perfil en la esquina 
	superior derecha de Trailhead -> clic en Hands-On Orgs (Organizaciones prácticas)
	-> iniciar.

Ver los metadatos que definen la estructura de una aplicación: Ir a determinada app -> 
	apartado properties -> selccionar alguna propiedad -> apartado details.

Restablecer la contraseña para su organización de trailhead playgroud: La integración entre 
	Trailhead y la organización de Playground hace que la contraseña predeterminada sea 
	inaccesible. Necesita una contraseña para acceder a la organización desde fuera de 
	Trailhead, por ejemplo desde herramientas de desarrollador, como Salesforce CLI y
	VS Code.
	Clic en Iniciador de aplicación para iniciar App Launcher -> busque y haga clic en 
	Playground Starter -> click en apartado get your login credentials -> reset my password.

AREA DE SETUP

Crear un objeto personalizado:  
	- Desde un archivo csv: En la parte sup derecha buscar el logo de settings -> 
		opción setup -> click apartado object manager -> click sobre créate ->      
		Seleccione Custom Object from Spreadsheet -> subir Excel.    
	- Desde la plataforma de Salesforce: En la parte sup derecha buscar el logo de 
		settings -> opción setup -> click apartado object manager -> click sobre 			créate -> custom object -> rellenar label y plural label con el nombre
		de interés (los campos Object Name (Nombre de objeto) y Record Name (Nombre 
		del registro) se rellenan automáticamente.) -> save -> modificar tab style
		al gusto -> next -> next -> save.        
		- Si se palomea la opción: 'Launch New Custom Tab Wizard after saving this 
			custom object' significa que Salesforce automáticamente crear una 
			pestaña (Custom Tab) para ese objeto.     

Agregar un campo a determinado objeto: En la parte sup derecha buscar el logo de settings -> 
	opción setup -> click apartado object manager -> click sobre determinado objeto -> 
	apartado izq field and relationships -> new -> seleccionar el tipo de dato del campo
	-> sig -> asginar el nombre del campo, su id y su description -> opcionalmente marcar
	si es obligatorio o no (required) -> next -> next -> save.

Comprobar la creación del campo nuevo para un objeto: En App Launcher (Iniciador de aplicación) 
	(Iniciador de 
	aplicación) -> buscar dreamhouse -> click sobre determinad objeto ->
	seleccionar un registro -> apartado details -> buscar el campo recién creado.

Ver todos los registros de un objeto: Ir a inicio -> buscar y seleccionar detemrinado objeto.
	- Para editar uno de los registros: se da click sobre determinado registro.

Crear una aplicación a un objeto: En la parte sup derecha buscar el logo de settings -> 
	opción setup -> buscar y seleccionar app manager -> new lighning app -> asignar los
	metadatos de la app -> estándar -> next -> next -> seleccionar el perfil system 
	administrator -> finish.

Crear un registro: En el inicio de aplicación -> buscar determinada app -> En los apartados
	estará determinado objeto -> new -> rellenar el registro.

crear una relación de búsqueda: En la parte sup derecha buscar el logo de settings -> 
	opción setup -> click apartado object manager -> click sobre determinado objeto -> 
	click en el apartado izq fields and relationships -> new -> lookup Relanship -> next
	-> elegir la tabla con la que se va a relacionar -> puro next.

crear una relación principal - detalle: En la parte sup derecha buscar el logo de settings -> 
	opción setup -> click apartado object manager -> click sobre determinado objeto -> 
	click en el apartado izq fields and relationships -> new -> master-detail relationship
	-> next -> elegir la tabla con la que se va a relacionar -> puro next.

Relacionar dos registros: En App Launcher (Iniciador de aplicación) -> buscar y seleccionar
	sales -> buscar en los apartados a determinado objeto -> click sobre determinado
	registro -> apartado related -> new 


SCHEMA BUILDER

Es una herramienta que permite diseñar, visualizar y modificar el modelo de datos.

Ingresar al schema builder: En la parte sup derecha buscar el logo de settings -> 
	opción setup -> Buscar en el lateral schema builder y dar click ahí.

Ver la relación entre ciertas tablas de interés: En el apartado izq buscar y presionar clear
	all -> seleccionar las tablas de interés -> presionar auto-layout.

Crear un objeto utilizando esta herramienta: En la barra lateral izquierda clic en la ficha 	Elements -> arrastrar el componente object -> introducir la información del objeto
	-> save.

Crear campos a determinado objeto utilizando esta herramienta: En la barra lateral izquierda clic 	en la ficha Elements -> arrastrar determinar tipo de campo a determinado objeto -> 
	introducir la info pertinente -> save.

A la hora de desarrollar en Salesforce Platform, existen numerosos lenguajes de programación que puede usar, por ejemplo:
	- El marco Lightning Web Components: un marco de trabajo de desarrollo de interfaz 
		de usuario basada en JavaScript.
	- Apex: lenguaje de programación creado por Salesforce con una sintaxis similar a Java.
	- Node.js: Entorno de ejecución JavaScript asíncrono y basado en eventos que se ha 
		diseñado para desarrollar aplicaciones de red ampliables.


COMPONENTS WEB LIGHNING

Desarrollo para dispositivos de escritorio y móviles basado en componentes para el desarrollo
de interfaces de usuario

Code builder: entorno de desarrollo integrado de preferencia de Salesforce que puede utilizar para desarrollar, depurar y probar código en su organización.


APEX

lenguaje de programación orientado a los objetos y con parámetros de asignación rigurosa que permite a los desarrolladores de Salesforce agregar lógica empresarial a eventos del sistema, como clics de botón y actualizaciones, a los registros relacionados.

Crear un nuevo proyecto de Salesforce y conectarlo a su organización de Trailhead Playground:
	*En vsc* ctrol + shift + p -> buscar y seleccionar SFDX: Create Project -> standard
	-> ingresar el nombre del proyecto -> ingresar la carpeta donde se estará el proeycto..

Para desarrollar con ápex, de acuerdo a la doc oficial, se recomienda tener vsc y la extensión
Salesforce Extension Pack (expanded). *

sf org login web -d -a nombreOrganizacion -> Para autenticarte en una organización de 
	Salesforce desde el navegador y configurarla con roles especiales dentro del CLI.

sf project deploy start -> Sirve para desplegar (deploy) el metadata de tu proyecto local 
	hacia una organización de Salesforce usando Salesforce CLI.
	Toma el contenido del proyecto Salesforce (normalmente la carpeta force-app/) y lo 
	envía a una org destino (scratch org, sandbox o producción).

.forceignore -> Para decirle a Salesforce CLI qué archivos o carpetas NO debe incluir cuando 
	haces operaciones como deploy, retrieve o convert de metadata.

sf apex generate class --name nombreClase --output-dir force-app/main/default/clases -> Para 
	crear una clase de ápex.
	Esto crear dos archivos: uno con extensión .cls y otro con extencion .cls-meta.xml
	en la ruta force-app/main/default.

Crear un script anonimo: es código de Apex que no se almacena en los metadatos, pero puede 	compilarse y ejecutarse.
	Crear un archivo con extensión .apex en la carpeta: scripts/ápex

*Dentro de una clase y donde se declaran atributos*
@InvocableVariable
tipoDato nombreAtributo; -> Para que este atributo pueda enviar datos al flujo.

*En la clase main*
@InvocableMethod
public static tipoDato nombreMetodo() {} -> Para que el método pueda invocarse en forma de 
	acción en el flujo.
