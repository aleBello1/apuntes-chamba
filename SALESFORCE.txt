Crear nuevo Trailhead Playground: Haga clic en su fotografía de perfil en la esquina 
	superior derecha de Trailhead -> clic en Hands-On Orgs (Organizaciones prácticas) ->
	clic en Create Playground (Crear Playground). El Playground puede tardar unos minutos
	en crearse.

Abrir determinado Trailhead Playground: Haga clic en su fotografía de perfil en la esquina 
	superior derecha de Trailhead -> clic en Hands-On Orgs (Organizaciones prácticas)
	-> iniciar.

Ver los metadatos que definen la estructura de una aplicación: Ir a determinada app -> 
	apartado properties -> selccionar alguna propiedad -> apartado details.

Restablecer la ontraseña ra su organización de trailhead playgroud: La integración entre 
	Trailhead y la organización de Playground hace que la contraseña predeterminada sea 
	inaccesible. Necesita una contraseña para acceder a la organización desde fuera de 
	Trailhead, por ejemplo desde herramientas de desarrollador, como Salesforce CLI y
	VS Code.
	Clic en Iniciador de aplicación para iniciar App Launcher -> busque y haga clic en 
	Playground Starter -> click en apartado get your login credentials -> reset my password.

AREA DE SETUP

Crear un objeto personalizado:  
	- Desde un archivo csv: En la parte sup derecha buscar el logo de settings -> 
		opción setup -> click apartado object manager -> click sobre créate ->      
		Seleccione Custom Object from Spreadsheet -> subir Excel.                                           

Agregar un campo a determinado objeto: En la parte sup derecha buscar el logo de settings -> 
	opción setup -> click apartado object manager -> click sobre determinado objeto -> 
	apartado izq field and relationships -> new -> seleccionar el tipo de dato del campo
	-> sig -> asginar el nombre del campo y su id -> next -> next -> save.

Comprobar la creación del campo nuevo para un objeto: En App Launcher (Iniciador de aplicación) 
	(Iniciador de 
	aplicación) -> buscar dreamhouse -> click sobre determinad objeto ->
	seleccionar un registro -> apartado details -> buscar el campo recién creado.

Ver el esquema de una aplicacion: En la parte sup derecha buscar el logo de settings -> 
	opción setup -> Buscar en el lateral schema builder y dar click ahí.

Ver todos los registros de un objeto: Ir a inicio -> buscar y seleccionar detemrinado objeto.
	- Para editar uno de los registros: se da click sobre determinado registro.

Crear una aplicación a un objeto: En la parte sup derecha buscar el logo de settings -> 
	opción setup -> buscar y seleccionar app mana ger -> new lighning app -> asignar los
	metadatos de la app -> estándar -> next -> next -> seleccionar el perfil system 
	administrator -> finish.

/////////////////////////////////DESARROLLO CON CODIGO//////////////////////////////////

A la hora de desarrollar en Salesforce Platform, existen numerosos lenguajes de programación que puede usar, por ejemplo:
	- El marco Lightning Web Components: un marco de trabajo de desarrollo de interfaz 
		de usuario basada en JavaScript.
	- Apex: lenguaje de programación creado por Salesforce con una sintaxis similar a Java.
	- Node.js: Entorno de ejecución JavaScript asíncrono y basado en eventos que se ha 
		diseñado para desarrollar aplicaciones de red ampliables.


COMPONENTS WEB LIGHNING

Desarrollo para dispositivos de escritorio y móviles basado en componentes para el desarrollo
de interfaces de usuario

Code builder: entorno de desarrollo integrado de preferencia de Salesforce que puede utilizar para desarrollar, depurar y probar código en su organización.


APEX

lenguaje de programación orientado a los objetos y con parámetros de asignación rigurosa que permite a los desarrolladores de Salesforce agregar lógica empresarial a eventos del sistema, como clics de botón y actualizaciones, a los registros relacionados.

Crear un nuevo proyecto de Salesforce y conectarlo a su organización de Trailhead Playground:
	*En vsc* ctrol + shift + p -> buscar y seleccionar SFDX: Create Project -> standard
	-> ingresar el nombre del proyecto -> ingresar la carpeta donde se estará el proeycto..

Para desarrollar con ápex, de acuerdo a la doc oficial, se recomienda tener vsc y la extensión
Salesforce Extension Pack (expanded). *

*Dentro de una clase y donde se declaran atributos*
@InvocableVariable
tipoDato nombreAtributo; -> Para que este atributo pueda enviar datos al flujo.

*En la clase main*
@InvocableMethod
public static tipoDato nombreMetodo() {} -> Para que el método pueda invocarse en forma de 
	acción en el flujo.


NODEJS

Entorno de ejecución JavaScript asíncrono y basado en eventos que se ha diseñado para 
desarrollar aplicaciones de red ampliables. 

force:project:create -> Para crear un proyecto nuevo.
